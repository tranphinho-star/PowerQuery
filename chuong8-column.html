Xử lý với Column trong Power Query

Thêm cột mới (Custom, Conditional, Index), thay đổi kiểu dữ liệu, Replace Values, Extract, và các thao tác quản lý cột.

8.1 Quản lý cột cơ bản

Power Query cung cấp nhiều thao tác quản lý cột trong tab Home và menu chuột phải:

Thao tácCách thực hiệnPhím tắt
Remove ColumnChọn cột → Home → Remove ColumnsDelete
Remove Other ColumnsChọn cột giữ lại → Home → Remove Other Columns—
Rename ColumnDouble-click header cộtF2
Duplicate ColumnChuột phải → Duplicate Column—
Move ColumnChuột phải → Move → Left/Right/Begin/End—
Reorder ColumnsKéo-thả header cột—
💡
Mẹo: Sử dụng Remove Other Columns thay vì Remove Columns khi bạn chỉ cần giữ lại vài cột. Điều này đảm bảo query vẫn hoạt động đúng khi nguồn dữ liệu có thêm cột mới.

8.2 Thay đổi Data Type (Kiểu dữ liệu)

Việc đặt đúng kiểu dữ liệu rất quan trọng để đảm bảo phép tính chính xác:

Cách thay đổi

  • Cách 1: Nhấp vào icon kiểu dữ liệu trên header cột (ABC, 123, 📅...)
  • Cách 2: Chọn cột → Transform → Data Type → chọn kiểu
  • Cách 3: Chuột phải → Change Type

Các kiểu dữ liệu phổ biến

IconKiểuMô tảVí dụ
ABCTextVăn bản, chuỗi ký tự"Nguyễn Văn A"
123Whole NumberSố nguyên42, -15, 0
1.2Decimal NumberSố thập phân3.14, 99.9
💲CurrencySố tiền tệ$1,234.56
📅DateNgày15/03/2024
🕐TimeGiờ14:30:00
📅🕐Date/TimeNgày + giờ15/03/2024 14:30
✅❌True/FalseLogic Booleantrue, false
⚠️
Lưu ý: Khi thay đổi kiểu dữ liệu, nếu có giá trị không thể chuyển đổi (ví dụ: "abc" → Number), ô đó sẽ hiển thị Error. Kiểm tra Column Quality để phát hiện lỗi.

8.3 Custom Column (Cột tùy chỉnh)

Tạo cột mới với công thức M tùy chỉnh — tương tự như thêm công thức trong Excel:

Cách tạo

  1. Tab Add Column → Custom Column
  2. Đặt tên cột mới
  3. Viết công thức M trong ô "Custom column formula"
  4. Tham chiếu cột bằng cú pháp: [TênCột]

Các ví dụ công thức

M Language - Custom Column
// Tính thành tiền
= [SoLuong] * [DonGia]

// Tính phần trăm
= [DoanhThu] / [TongDoanhThu] * 100

// Gộp chuỗi
= [Ho] & " " & [Ten]

// Tính toán với ngày
= Date.Year([NgayDat])

// Chuỗi ký tự
= Text.Upper([HoTen])

// Kiểm tra điều kiện
= if [Tuoi] >= 18 then "Trưởng thành" else "Vị thành niên"

8.4 Conditional Column (Cột điều kiện)

Tạo cột mới dựa trên điều kiện if-then-else với giao diện trực quan (không cần viết code):

Cách tạo

  1. Tab Add Column → Conditional Column
  2. Đặt tên cột mới
  3. Thiết lập các điều kiện:
    • Column name: cột để kiểm tra
    • Operator: equals, contains, begins with, is greater than...
    • Value: giá trị so sánh
    • Output: giá trị kết quả khi điều kiện đúng
  4. Nhấn Add Clause để thêm điều kiện (else if)
  5. Thiết lập giá trị Otherwise (else)

Ví dụ: Phân loại khách hàng

Điều kiệnOutput
Nếu [DoanhThu] ≥ 100.000.000"VIP"
Nếu [DoanhThu] ≥ 50.000.000"Gold"
Nếu [DoanhThu] ≥ 10.000.000"Silver"
Otherwise"Standard"

8.5 Index Column (Cột chỉ mục)

Thêm cột đánh số thứ tự tự động:

  • Tab Add Column → Index Column
  • Tùy chọn:
    • From 0: Bắt đầu từ 0 (0, 1, 2...)
    • From 1: Bắt đầu từ 1 (1, 2, 3...)
    • Custom: Chọn giá trị bắt đầu và bước nhảy

8.6 Column from Examples

Tính năng thông minh — Power Query sẽ tự đoán công thức dựa trên ví dụ bạn nhập:

  1. Tab Add Column → Column from Examples
  2. Chọn From All Columns hoặc From Selection
  3. Nhập giá trị mẫu vào cột mới cho 2-3 dòng đầu
  4. Power Query sẽ tự xác định pattern và điền giá trị cho các dòng còn lại
  5. Nếu đúng → nhấn OK. Nếu sai → nhập thêm ví dụ để tinh chỉnh
💡
Ứng dụng tuyệt vời: Trích xuất năm từ ngày, phần đầu của email, tạo mã tự động, format số điện thoại... mà không cần nhớ cú pháp M language!

8.7 Replace Values (Thay thế giá trị)

Tìm và thay thế giá trị trong cột:

  1. Chọn cột → Transform → Replace Values
  2. Nhập giá trị cần tìm (Value to Find) và giá trị thay thế (Replace With)

Mẹo hữu ích

  • Thay thế null bằng giá trị mặc định: Value to Find = null
  • Thay thế chuỗi rỗng: Value to Find = "" (chuỗi rỗng)
  • Xóa nội dung: Replace With = "" (chuỗi rỗng)

8.8 Extract (Trích xuất)

Trích xuất phần dữ liệu cụ thể từ cột text:

Chức năng ExtractMô tảVí dụ
LengthĐộ dài chuỗi"Hello" → 5
First CharactersN ký tự đầu"Hello" (3) → "Hel"
Last CharactersN ký tự cuối"Hello" (2) → "lo"
RangeKý tự từ vị trí X, N ký tự"Hello" (1,3) → "ell"
Text Before DelimiterText trước ký tự phân cách"user@email.com" (@) → "user"
Text After DelimiterText sau ký tự phân cách"user@email.com" (@) → "email.com"
Text Between DelimitersText giữa 2 ký tự phân cách"(HCM)" → "HCM"

Truy cập: Tab Transform → Extract hoặc Tab Add Column → Extract

📋
Khác biệt: Extract từ tab Transform sẽ thay thế cột gốc. Extract từ tab Add Column sẽ tạo cột mới và giữ nguyên cột gốc.

8.9 Using Locale (Định dạng theo vùng)

Khi dữ liệu đến từ các quốc gia khác nhau, định dạng ngày và số có thể khác nhau. Using Locale giúp chuyển đổi chính xác:

Cách sử dụng

  1. Chọn cột → Right-click → Change Type → Using Locale...
  2. Chọn Data Type và Locale (ngôn ngữ vùng)

Ví dụ định dạng theo Locale

Giá trị gốc (text)LocaleKết quả
"03/15/2024"English (US)15/03/2024 ✅
"15.03.2024"German15/03/2024 ✅
"1.234,56"Vietnamese1234.56 ✅
"1,234.56"English (US)1234.56 ✅
M Language - Using Locale
// Chuyển text thành Date với locale Mỹ (MM/DD/YYYY)
= Table.TransformColumnTypes(
    PreviousStep,
    {{"NgayDat", type date}},
    "en-US"  // Locale
)

// Chuyển text thành Number với locale Việt Nam (dấu phẩy thập phân)
= Table.TransformColumnTypes(
    PreviousStep,
    {{"GiaTri", type number}},
    "vi-VN"  // Locale
)

8.10 Column Quality, Distribution & Profile

Ba công cụ phân tích chất lượng dữ liệu cực kỳ hữu ích trong tab View:

Column Quality

Hiển thị thanh phần trăm ở đầu mỗi cột với 3 trạng thái:

  • Valid (Xanh): Giá trị hợp lệ
  • Error (Đỏ): Giá trị bị lỗi
  • Empty (Xám): Giá trị trống/null

Column Distribution

Hiển thị biểu đồ phân bố giá trị cho mỗi cột:

  • Distinct: Số giá trị khác nhau
  • Unique: Số giá trị xuất hiện đúng 1 lần
  • Bar chart trực quan cho thấy sự phân bố

Column Profile

Thông tin thống kê chi tiết cho cột đang chọn:

  • Cột Text: Count, Empty, Distinct, Unique, Max/Min Length, giá trị phổ biến nhất
  • Cột Number: Count, Error, Min, Max, Average, Standard Deviation, Count of Zero, Even, Odd
  • Cột Date: Count, Earliest, Latest
⚠️
Lưu ý quan trọng: Mặc định, Column Profile chỉ phân tích 1,000 dòng đầu tiên. Để phân tích toàn bộ dữ liệu, nhấp vào link trên Status Bar: "Column profiling based on top 1000 rows" → chọn "Based on entire dataset".

Bật tính năng: Tab View → ☑️ Column Quality, ☑️ Column Distribution, ☑️ Column Profile

8.11 Error Types chi tiết

Power Query phân loại lỗi theo nguồn gốc. Hiểu Error Types giúp xử lý chính xác:

Error TypeNguyên nhânVí dụ
Expression.Error Lỗi trong biểu thức hoặc tính toán Chia cho 0, truy cập cột không tồn tại
DataSource.Error Không thể kết nối hoặc đọc nguồn File bị xóa, server ngắt, quyền truy cập
DataFormat.Error Dữ liệu không đúng định dạng mong đợi Text "abc" chuyển thành Number
Formula.Firewall Conflict Privacy Levels giữa các nguồn Kết hợp Public + Private data source

8.12 Error Handler Pattern nâng cao

Dùng try để bắt lỗi và lấy thông tin chi tiết:

M Language - Error Handler nâng cao
// try trả về Record với 2 trường: HasError và Value
let
    result = try Number.From([GiaTri])
in
    if result[HasError] then
        "Lỗi: " & result[Error][Message]
    else
        result[Value]

// Pattern: Kiểm tra lỗi từng cột
= Table.AddColumn(Source, "Status",
    each if try [SoLuong] * [DonGia] then "OK"
         else "Error: " & (try [SoLuong] * [DonGia])[Error][Message]
)

// Pattern: Log lỗi ra file
= Table.AddColumn(Source, "ErrorLog",
    each let r = try [CotTinhToan]
         in if r[HasError]
            then [Reason = r[Error][Reason], Message = r[Error][Message]]
            else null
)
💡
Best Practice: Khi xử lý dữ liệu từ nguồn bên ngoài, luôn bọc các phép tính quan trọng bằng try...otherwise để tránh toàn bộ query bị fail vì 1 dòng lỗi.

8.13 Remove Columns by Pattern (Xóa cột theo tên mẫu)

Khi import dữ liệu có nhiều cột thừa với tên theo pattern (vd: "Column1", "Column2"...), xóa hàng loạt bằng M:

M Language - Remove Columns by Pattern
// Xóa tất cả cột có tên bắt đầu bằng "Column"
let
    AllCols = Table.ColumnNames(Source),
    ColsToRemove = List.Select(AllCols,
        each Text.StartsWith(_, "Column")),
    Result = Table.RemoveColumns(Source, ColsToRemove)
in Result

// Xóa cột chứa từ "temp" hoặc "test"
let
    AllCols = Table.ColumnNames(Source),
    ColsToRemove = List.Select(AllCols,
        each Text.Contains(Text.Lower(_), "temp")
          or Text.Contains(Text.Lower(_), "test")),
    Result = Table.RemoveColumns(Source, ColsToRemove)
in Result

// CHỈ GIỮ cột có tên nằm trong danh sách
let
    ColsToKeep = {"MaNV", "HoTen", "PhongBan", "Luong"},
    Result = Table.SelectColumns(Source, ColsToKeep)
in Result
💡
Pro tip: Kết hợp với Table.ColumnNames + List.Select để xử lý cột động — rất hữu ích khi số cột thay đổi giữa các lần refresh.

8.14 Data Type Best Practices

Kiểu dữ liệu là nguồn lỗi phổ biến nhất trong Power Query. Các best practices:

Auto-Detect Pitfalls (bẫy tự nhận dạng)

⚠️
Cẩn thận với "Changed Type" tự động! Power Query thường tự thêm bước "Changed Type" dựa trên 200 dòng đầu tiên. Điều này có thể sai nếu:
  • 200 dòng đầu toàn số, nhưng dòng 201+ có text → lỗi chuyển đổi
  • Ngày format "01/02/2024": là 1/2 hay 2/1? (phụ thuộc Locale)
  • Số "1.000" là 1 hay 1000? (phụ thuộc dấu phân cách)

Using Locale (chỉ định vùng miền)

Khi data có format khác Locale mặc định, dùng Using Locale để parse đúng:

M Language - Using Locale
// Ngày Mỹ (MM/DD/YYYY) trên máy VN
= Table.TransformColumnTypes(Source,
    {{"Ngay", type date}}, "en-US")

// Ngày Anh (DD/MM/YYYY)
= Table.TransformColumnTypes(Source,
    {{"Ngay", type date}}, "en-GB")

// Số Đức (1.000,50 = một nghìn phẩy năm mươi)
= Table.TransformColumnTypes(Source,
    {{"Gia", type number}}, "de-DE")

Checklist kiểu dữ liệu

LoạiKiểm traBest Practice
DateFormat DD/MM hay MM/DD?Luôn chỉ định Locale
NumberDấu phẩy hay chấm?Dùng Using Locale cho data quốc tế
TextEncoding đúng? UTF-8/ANSI?Xác nhận encoding khi import
Boolean0/1, True/False, Yes/No?Chuẩn hóa trước khi chuyển type

8.15 Named Ranges & Dynamic Arrays từ Power Query

Kết hợp Power Query output với Dynamic Arrays (Excel 365) để tạo dữ liệu linh hoạt:

Tạo Named Range từ PQ Table

  1. Load PQ → Excel Table (VD: tên Table = tbl_SanPham)
  2. Formulas → Name Manager → New
  3. Name: rng_DanhMuc, Refers to: =UNIQUE(tbl_SanPham[DanhMuc])

→ Named Range tự cập nhật khi Refresh Power Query!

Dynamic Arrays kết hợp PQ

Excel Formulas (365)
// Danh sách unique từ PQ table
= UNIQUE(tbl_DonHang[TenSP])

// Lọc + Sắp xếp từ PQ output
= SORT(FILTER(tbl_DonHang,
    tbl_DonHang[Thang]=MONTH(TODAY())),
    6, -1)  // Sort theo cột 6 giảm dần

// Top 5 sản phẩm bán chạy
= TAKE(SORT(UNIQUE(
    CHOOSECOLS(tbl_DonHang, 3, 6)),  // TenSP, ThanhTien
    2, -1), 5)  // Sort col 2 desc, lấy 5

// Dropdown Data Validation
// Data → Validation → List → =rng_DanhMuc
💡
Power Query xử lý ETL (nặng), Dynamic Arrays xử lý trình bày (nhẹ). Kết hợp cả 2 = giải pháp hoàn hảo.

8.16 Column from Examples (Tạo cột từ ví dụ)

Power Query có thể tự đoán công thức khi bạn chỉ cần nhập vài ví dụ:

Cách dùng

  1. Add Column → Column from Examples → From All Columns
  2. Nhập 1-3 giá trị mẫu ở cột mới
  3. Power Query tự nhận dạng pattern và điền toàn bộ
  4. Kiểm tra kết quả → OK

Ví dụ thực tế

Dữ liệu gốcBạn nhập ví dụPQ tự tạo
"Nguyễn Văn An"nhập: "NV An"Tự tạo: rút gọn Họ + Tên
"2024-03-15"nhập: "Tháng 03/2024"Tự tạo: format lại ngày
"an.nguyen@abc.com"nhập: "abc.com"Tự tạo: extract domain
"SP-001-VN"nhập: "001"Tự tạo: extract mã giữa
💡
Khi nào dùng Column from Examples? Khi bạn biết kết quả mong muốn nhưng chưa biết hàm M nào cần dùng. PQ sẽ viết công thức cho bạn — bạn có thể xem và học từ đó!

Bài tập 8.1: Custom Column — Tính toán

Mục tiêu: Tạo cột tính toán với công thức M

  1. Tạo bảng:
    TenSPGiaNhapGiaBanSoLuong
    Laptop Dell120000001500000010
    Chuột Logitech25000035000050
    Bàn phím Corsair800000120000030
  2. Mở Power Query
  3. Add Column → Custom Column → tạo cột "DoanhThu": [GiaBan] * [SoLuong]
  4. Add Column → Custom Column → tạo cột "LoiNhuan": ([GiaBan] - [GiaNhap]) * [SoLuong]
  5. Add Column → Custom Column → tạo cột "TyLeLoi": ([GiaBan] - [GiaNhap]) / [GiaNhap] * 100
  6. Đổi kiểu dữ liệu cột TyLeLoi → Decimal Number
  7. Close & Load

Bài tập 8.2: Conditional Column — Phân loại

Mục tiêu: Sử dụng Conditional Column để phân nhóm

  1. Tạo bảng nhân viên:
    HoTenPhongBanLuongNamKinhNghiem
    Nguyễn Văn AIT250000008
    Trần Thị BKD150000003
    Lê Văn CIT3500000012
    Phạm Thị DHR120000001
    Hoàng Văn EKD200000005
  2. Mở Power Query
  3. Add Column → Conditional Column → tạo cột "CapBac":
    • Nếu NamKinhNghiem ≥ 10 → "Senior"
    • Nếu NamKinhNghiem ≥ 5 → "Mid-level"
    • Nếu NamKinhNghiem ≥ 2 → "Junior"
    • Otherwise → "Fresher"
  4. Add Column → Conditional Column → tạo cột "MucLuong":
    • Nếu Luong ≥ 30.000.000 → "Cao"
    • Nếu Luong ≥ 15.000.000 → "Trung bình"
    • Otherwise → "Thấp"
  5. Close & Load

Bài tập 8.3: Extract & Replace — Xử lý email

Mục tiêu: Thực hành Extract, Replace Values, Column from Examples

  1. Tạo bảng:
    HoTenEmailSoDienThoai
    Nguyễn Văn Anan.nguyen@company.com+84-912-345-678
    Trần Thị Bìnhbinh.tran@gmail.com+84-987-654-321
    Lê Hoàng Châuchau.le@company.com+84-376-123-456
  2. Mở Power Query
  3. Extract username từ email: Add Column → Extract → Text Before Delimiter → "@"
  4. Extract domain từ email: Add Column → Extract → Text After Delimiter → "@"
  5. Replace +84 thành 0: Chọn cột SoDienThoai → Transform → Replace Values → "+84" thành "0"
  6. Column from Examples: Thử tạo cột "TenVietTat" bằng cách nhập ví dụ "NVA" cho dòng đầu
  7. Close & Load
